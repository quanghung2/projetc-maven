Guides

When there is an incoming call, the ticket will be created with the following info (using POST)
  • Virtual Line
  • Incoming number
  • Access number
When the call finishes, the same ticket will be updated with the following info (using PUT)
  • Virtual Line
  • Incoming number
  • Access number
  • Working hour type
  • Date & time
  • Pressed digits
  • Endpoint
  • Call recording link
Please take note that in order to receive the ticket update, the ticket ID or reference has to be returned in the first HTTP Post.
